## Detalhes técnicos que a próxima sessão precisa saber
- **Célula**: backend `POST /celulas` deriva `diaSemana` de `weekdayDaString(dataPrimeiroEncontro)` e grava `dataUtcDaString(...)` (UTC-pinada, TZ-safe). O front manda wall-clock ingênuo "YYYY-MM-DDTHH:mm" (`montarPayloadCelula`, sem `toISOString`). **Não reintroduzir validação de coerência que rejeite.**
- **Status do usuário** (`lib/papeis.js` `statusDeUsuario`): PENDENTE=`!aprovado&&ativo`; REPROVADO=`!aprovado&&!ativo`; INATIVO=`aprovado&&!ativo`; ATIVO=`aprovado&&ativo`. `recusar` faz soft (usuarios.js). Pendentes filtra `ativo:true`.
- **Migração `cep`**: `ADD COLUMN IF NOT EXISTS` (idempotente). Se o entrypoint falhar com P3009 (migration falha), limpar: `DELETE FROM _prisma_migrations WHERE finished_at IS NULL;` (parar o container antes) e reiniciar.
- **DB do host**: só containers na rede-docker autenticam; do host macOS use `docker exec icelula-db psql -U icelula -d icelula`.
